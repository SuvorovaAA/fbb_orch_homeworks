## Домашнее задание 1: описание файлов

### `complement.py`
Игрушечный скрипт, который получает на вход последовательность и возвращает её reverse-complement и 
значение GC-состава с точностью до третьего знака после запятой.

Пример использования:

```bash
./complement.py --seq ATGACT
AGTCAT
0.333
```

### `test.fna`
Текстовый файл, содержащий примеры аминокислотных последовательностей, 
которые можно использовать для запуска скрипта `count_kmers.py`

### `count_kmers.py`
Исходно написала скрипт так, что можно задать имя `.json` файла с выдачей, 
установила значение по умолчанию `cnts.json`. Аналогично можно передать
значение `k`, значение по умолчанию установила 4.

На github изменила значение по умолчанию на 2, а в локальной версии за 
неимением идей, что бы ещё вытворить, добавила пару строк с комментарием. 
При попытке сделать `push` локальной версии естественно получила ошибку:

```bash
$ git push origin main
To github.com:SuvorovaAA/fbb_orch_homeworks.git
 ! [rejected]        main -> main (fetch first)
error: failed to push some refs to 'github.com:SuvorovaAA/fbb_orch_homeworks.git'
```

Насколько я поняла, в этом случае принято делать pull, локально разрешать конфликт,
после чего снова push'ить. Для слияния выбрала стратегию `--rebase`:

```bash
$ git pull --rebase origin main
```

В полученной версии файла сохранились и измененное дефолтное значение `k`, и 
новые строчки. Вроде как после этого конфликт разрешился, но почему-то git не хотел 
воспринимать получившийся файл как измененный, из-за чего не хотел добавлять его в коммит.
Почему это именно так работает, я так и не поняла, честно говоря.
